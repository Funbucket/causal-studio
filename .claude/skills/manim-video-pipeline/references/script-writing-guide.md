# 스크립트 작성 가이드

Scene 구조 기반 내레이션 스크립트 작성 가이드.

## 목표

- Scene 구조(`src/scene_outline.md`)를 그대로 유지
- 씬별 스크립트 파일 생성
- TTS(ElevenLabs/Piper)로 mp3 변환 가능한 형식

## 출력 구조

```
src/scripts/
├── 01_{scene_name}.txt
├── 02_{scene_name}.txt
├── 03_{scene_name}.txt
└── ...
```

## 파일명 규칙

- Scene 순서 prefix: `01_`, `02_`, ...
- snake_case 유지
- src/scene_outline.md의 Scene 이름과 일치

## 스크립트 작성 규칙

### 분량
- 1~4문장 (가능하면 1~3문장)
- 문장은 짧게
- 줄바꿈으로 호흡 표현

### 톤
- 차분하고 교육적인 톤
- 과장된 감정 표현 금지
- 친근하지만 전문적

### 피해야 할 것
- 메타 설명 ("이제 다음 장면에서...", "여기서 보시면...")
- 초 단위 시간 표현 금지
- 영상에서 이미 명확한 시각 요소 장황하게 반복
- "자, 그럼", "네," 같은 불필요한 추임새

## 작성 전략

각 Scene 스크립트 흐름:

1. **문제 제기**: Scene의 핵심 질문
2. **오해 유도**: Expected Misconception 자연스럽게 드러내기
3. **Visual Pivot 직전**: 긴장 형성
4. **개념 정리**: 간결한 결론

단, 모든 요소를 다 넣을 필요 없음. 영상 리듬에 맞게 밀도 조절.

## 예시

### Scene 01: paradox_of_comparison
```
약이 입원 기간을 줄여주는지 알아봅시다.
단순히 평균을 비교해보면,
치료받은 환자가 하루 정도 빨리 퇴원했네요.
그런데 이게 약의 진짜 효과일까요?
```

### Scene 02: confounding_revealed
```
두 그룹이 정말 비슷할까요?
알고 보니, 중증도가 다릅니다.
중증 환자는 약을 더 많이 받고,
중증일수록 입원도 길어지죠.
이런 변수를 교란 변수라고 합니다.
```

### Scene 06: matching_intuition
```
그럼 비슷한 사람끼리만 비교하면 어떨까요?
같은 나이, 같은 소득 수준.
이 방법을 매칭이라고 합니다.
```

## 품질 체크리스트

- [ ] Scene 순서/이름이 outline과 일치하는가?
- [ ] 문장이 충분히 짧은가?
- [ ] 줄바꿈이 호흡을 반영하는가?
- [ ] 메타 설명이 없는가?
- [ ] 영상과 중복 설명이 없는가?
- [ ] TTS로 자연스럽게 읽힐 수 있는가?

## 주의사항

- Scene 구조를 변경하지 말 것
- Scene 개수, 순서, 이름 유지
- 타임코드(초 단위) 작성 금지
- 영상 길이 추정 금지
- 코드 작성 금지 (이 단계에서)
