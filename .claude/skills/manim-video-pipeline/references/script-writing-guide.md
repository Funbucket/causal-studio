# 스크립트 작성 가이드

## 파일 구조

```
src/scripts/
├── 01_{scene_name}.txt
├── 02_{scene_name}.txt
└── ...
```

파일명: `{NN}_{scene_name}.txt` — Scene 클래스 이름과 일치.

## 작성 규칙

**분량**: 원본 ipynb의 논리 전개·예시·수식 해석을 보존. 줄이는 대상은 중복 표현이지 개념 설명이 아님.

**톤**: 차분하고 교육적. 과장된 감정·추임새 금지 ("자, 그럼", "네,").

**피해야 할 것**:
- 메타 설명 ("이제 다음 장면에서...", "여기서 보시면...")
- 초 단위 시간 표현
- 화면에 이미 명확한 요소를 장황하게 반복
- 수식 의미 설명 생략, 예시 맥락·부호 해석 생략

## 흐름 구조 (Scene별)

1. **문제 제기** — Scene의 핵심 질문
2. **오해 유도** — Expected Misconception 자연스럽게 드러내기
3. **Visual Pivot 직전** — 긴장 형성
4. **개념 정리** — 간결한 결론

모든 요소를 다 넣을 필요 없음. 리듬에 맞게 밀도 조절.

## 예시

```
# Scene 01: policy_choice_intro
예산이 한정되어 있다고 해봅시다.
태블릿을 지급하는 편이 좋을까요?
아니면 도서관과 학습 공간을 확충하는 편이 좋을까요?
정말 알고 싶은 건 어느 선택이 학업 성취도를 높이느냐는 점입니다.

# Scene 02: repeated_causal_questions
이런 질문은 교육 정책에서만 나오지 않습니다.
교육 수준이 높아지면 소득이 늘어날까요?
담배 가격을 올리면 흡연율이 줄어들까요?
광고 캠페인은 실제로 매출을 끌어올릴까요?
```

## TTS 친화적 수식 표기

스크립트 파일은 ElevenLabs TTS가 읽는 텍스트다. 수식 기호·LaTeX·서브스크립트는 그대로 두면 읽히지 않거나 엉뚱하게 발음된다. 아래 규칙으로 모두 한국어 발음 표기로 교체한다.

### 변환 원칙
- **화면에 수식이 나오더라도 스크립트에는 발음대로 쓴다.** 수식 설명은 화면이 담당하고, 음성은 의미를 말한다.
- 첨자(Y_0, Y_1)는 "와이 영", "와이 일"처럼 쓴다.
- 조건부 기댓값 `E[Y_0|T=1]`는 "처치 집단의 와이 영 기댓값" 또는 "티가 1일 때 와이 영의 기댓값"으로 풀어 쓴다.
- 영문 약어(ATE, ATT, ATC, RCT, ITE)는 한글 발음으로 쓴다 — "에이티이", "에이티티", "에이티씨", "알씨티", "아이티이".
- 수식 연산자(⊥, →, ∑, =)는 "독립", "이면", "합", "같아집니다"처럼 문장으로 풀어 쓴다.

### 변환 테이블

| 수식 표기 | 스크립트 표기 |
|-----------|--------------|
| `Y_0`, `Y₀` | 와이 영 |
| `Y_1`, `Y₁` | 와이 일 |
| `T_i` | 티 아이 |
| `E[Y_0|T=1]` | 처치 집단의 와이 영 기댓값 |
| `E[Y_0|T=0]` | 비교 집단의 와이 영 기댓값 |
| `E[Y_1|T=1]` | 처치 집단의 와이 일 기댓값 |
| `E[Y_1\|T=0]` | 비교 집단의 와이 일 기댓값 |
| `E[Y_1-Y_0\|T=1]` | 처치 집단의 평균 처치 효과 |
| `(Y_0,Y_1)⊥T` | 와이 영과 와이 일이 티와 독립 |
| `ATT` | 에이티티 |
| `ATC` | 에이티씨 |
| `ATE` | 에이티이 |
| `ITE` | 아이티이 |
| `RCT` | 알씨티 |
| `Bias = 0` | 편향이 0 |
| `gold standard` | 골드 스탠다드 |
