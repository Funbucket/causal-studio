# 스크립트 작성 가이드

Scene 구조 기반 내레이션 스크립트 작성 가이드.

## 목표

- 현재 작업 Scene 하나를 기준으로 스크립트 파일 생성
- 씬별 스크립트 파일 생성
- TTS(ElevenLabs/Piper)로 mp3 변환 가능한 형식
- 원본 ipynb의 개념 설명, 예시, 수식 해석을 최대한 보존
- 기본은 현재 Scene 하나만 작성하고 사용자 확인 후 다음 Scene으로 이동
- 코드 docstring/주석에 적힌 참고 3b1b 영상의 전개 리듬도 반영

## 출력 구조

```
src/scripts/
├── 01_{scene_name}.txt
├── 02_{scene_name}.txt
├── 03_{scene_name}.txt
└── ...
```

## 파일명 규칙

- Scene 순서 prefix: `01_`, `02_`, ...
- snake_case 유지
- 현재 작업 중인 Scene 클래스 이름/주제와 일치

## 스크립트 작성 규칙

### 분량
- 기본은 "충분히 자세하게" 쓴다
- 짧게 잘라 요약하는 것보다, 원본의 설명 논리를 보존하는 것을 우선한다
- Scene당 여러 문단이 되어도 괜찮다
- 문장은 너무 길게 꼬지 말고, 줄바꿈으로 호흡을 나눈다

### 톤
- 차분하고 교육적인 톤
- 과장된 감정 표현 금지
- 친근하지만 전문적

### 피해야 할 것
- 메타 설명 ("이제 다음 장면에서...", "여기서 보시면...")
- 초 단위 시간 표현 금지
- 영상에서 이미 명확한 시각 요소 장황하게 반복
- "자, 그럼", "네," 같은 불필요한 추임새
- 원본 ipynb의 핵심 설명을 한두 문장으로 뭉개는 과도한 생략
- 수식은 보여주는데 그 의미 설명을 스크립트에서 빼버리는 것
- 예시의 맥락, 부호 해석, 왜 중요한지에 대한 설명 생략

## 작성 전략

각 Scene 스크립트 흐름:

1. **문제 제기**: Scene의 핵심 질문
2. **오해 유도**: Expected Misconception 자연스럽게 드러내기
3. **Visual Pivot 직전**: 긴장 형성
4. **개념 정리**: 간결한 결론

단, 모든 요소를 다 넣을 필요 없음. 영상 리듬에 맞게 밀도 조절.

## ipynb 충실 반영 원칙

- markdown cell의 핵심 논리와 설명 순서를 먼저 따른다.
- 새 Scene를 쓰기 전에 반드시 아래를 먼저 확인한다.
  - 직전 Scene의 마지막 문장
  - 현재 Scene의 첫 문장
  - ipynb에서 이 Scene가 담당하는 셀/문단 범위
- Scene 시작 문장은 ipynb 원문에서 실제로 새 설명이 시작되는 지점이어야 한다.
- Scene 경계는 설명이 자연스럽게 끊기는 지점에 둔다. 개념 정의 중간, 예시 해석 중간, 수식 의미 설명 중간에서 자르지 않는다.
- "정의 -> 의미 설명", "예시 제시 -> 왜 중요한지 해석"은 가능하면 같은 Scene 안에 둔다.
- 원본에 중요한 수식이 있으면, 스크립트에도 "수식을 읽어주는 문장"이 아니라 "그 수식이 뜻하는 바"를 포함한다.
- 원본 예시가 태블릿, 학교 재정, 온라인 수업처럼 서사의 핵심이면 반드시 남긴다.
- 계산 결과가 중요한 경우 부호와 크기의 해석까지 말한다.
- 관찰 데이터의 한계, bias의 의미, RCT의 역할처럼 원본의 주장 연결고리를 유지한다.
- Scene은 영상 친화적으로 다듬되, 원본 글을 빈약한 요약본으로 축소하지 않는다.

## 예시

### Scene 01: policy_choice_intro
```
예산이 한정되어 있다고 해봅시다.
태블릿을 지급하는 편이 좋을까요?
아니면 도서관과 학습 공간을 확충하는 편이 좋을까요?
정말 알고 싶은 건 어느 선택이 학업 성취도를 높이느냐는 점입니다.
```

### Scene 02: repeated_causal_questions
```
이런 질문은 교육 정책에서만 나오지 않습니다.
교육 수준이 높아지면 소득이 늘어날까요?
담배 가격을 올리면 흡연율이 줄어들까요?
광고 캠페인은 실제로 매출을 끌어올릴까요?
```

## 품질 체크리스트

- [ ] Scene 순서/이름이 현재 작업 코드와 일치하는가?
- [ ] 직전 Scene의 마지막 문장과 현재 Scene의 첫 문장이 자연스럽게 이어지는가?
- [ ] 현재 Scene의 시작/끝이 ipynb 원문 경계와 맞는가?
- [ ] 원본 ipynb의 핵심 설명이 빠지지 않았는가?
- [ ] 중요한 예시와 수식 해석이 유지되었는가?
- [ ] 과도한 요약으로 개념이 빈약해지지 않았는가?
- [ ] 줄바꿈이 호흡을 반영하는가?
- [ ] 메타 설명이 없는가?
- [ ] 영상과 중복 설명이 없는가?
- [ ] TTS로 자연스럽게 읽힐 수 있는가?
- [ ] ElevenLabs로 바로 읽었을 때도 호흡이 너무 깨지지 않는가?
- [ ] 긴 장면이라면 문단 단위 생성으로 나눠도 연결이 자연스러운가?

## 주의사항

- Scene 구조를 변경하지 말 것
- Scene 개수, 순서, 이름 유지
- 타임코드(초 단위) 작성 금지
- 영상 길이 추정 금지
- 코드 작성 금지 (이 단계에서)
- 사용자가 확인하지 않은 다음 Scene 스크립트를 한꺼번에 생성하지 말 것
- 참고 3b1b 영상이 있으면 그 영상의 호흡, 전환 타이밍 감각, 강조 순서를 참고하되 문장을 베끼지 말 것
